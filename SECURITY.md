# Security Policy

Thank you for helping improve the security of *mORMot 2*.

If you believe you have discovered a security vulnerability, please report it privately so that the issue can be investigated and, if necessary, fixed before public disclosure.

## Reporting a Vulnerability

Please send your report by email to:

**security [at] synopse [dot] info**

(replace `[at]` with `@` and `[dot]` with `.`)

Please do **not** report security vulnerabilities through public GitHub issues or discussions.

If possible, include:

- the affected *mORMot 2* version (or Git commit);
- the target platform and compiler;
- a description of the vulnerability;
- steps to reproduce the issue;
- proof-of-concept code, if available;
- your assessment of the potential impact;
- any useful information or proposal to fix the issue.

## What to Expect

As *mORMot 2* is maintained by a small team of developers, responses may occasionally be delayed.

A best effort will be made to:

- acknowledge receipt of the report within a few days;
- investigate and reproduce the issue;
- work with the reporter to prepare an appropriate fix;
- coordinate public disclosure after a fix is available whenever practical.

## Supported Versions

Security fixes are generally provided for:

- the current development branch;
- the latest stable release.

Older releases may not receive security updates.
Especially, the deprecated *mORMot 1* repository is likely not to receieve any patch.

## Scope

Security reports are especially welcome for issues affecting:

- remote code execution;
- authentication or authorization bypass;
- cryptographic implementation flaws;
- information disclosure;
- privilege escalation;
- denial of service caused by malformed or malicious input.

General bugs, feature requests, and usability issues should continue to be reported through the normal GitHub issue tracker.

Thank you for helping keep *mORMot 2* secure!
